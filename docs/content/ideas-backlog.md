# Content Ideas Backlog

Use this document for raw website content ideas that are worth keeping but are
not ready for active work. GitHub issues should be created when an idea is ready
for drafting, review, or publication work with acceptance criteria.

## Workflow

1. Add rough ideas here with source links and a short note about the angle.
2. Keep ideas lightweight; do not turn this file into a drafting workspace.
3. When an idea is ready to work, create a GitHub issue and link back to the
   relevant entry in this document.
4. Close or update the issue when the content ships, but keep this document as
   the long-term idea archive.
